Why Public-Key Cryptography? — Slides 3–5

قبل عام ١٩٧٦، كل التشفير كان Symmetric (متماثل) — يعني نفس المفتاح للتشفير وفك التشفير. هذا سبّب مشكلتين كبيرتين:

Before 1976, all encryption was Symmetric — same key for both encryption and decryption. This caused two major problems:

  • Key Distribution Problem (مشكلة توزيع المفاتيح): كيف تُوصّل المفتاح السري لأول مرة بأمان؟ إذا القناة غير آمنة، المفتاح ينكشف.

    Key Distribution Problem: How do you securely deliver the secret key for the first time? If the channel is insecure, the key is exposed.

  • Digital Signature Problem (مشكلة التوقيع الرقمي): كيف تُثبت أن رسالة جاءت منك فعلاً وليس من شخص آخر؟

    Digital Signature Problem: How do you prove a message truly came from you and not someone impersonating you?

في ١٩٧٦، حل Diffie و Hellman من Stanford كلتا المشكلتين باستخدام مفتاحين مختلفين رياضياً — Asymmetric Encryption (التشفير غير المتماثل).

In 1976, Diffie and Hellman from Stanford solved both problems using two mathematically linked keys — Asymmetric Encryption.

💡 المبدأ الأساسي
💡 Core Principle

Public Key (مفتاح عام): يعرفه الجميع — للتشفير فقط.
Private Key (مفتاح خاص): تعرفه أنت فقط — لفك التشفير.

Public Key: Known to everyone — used for encryption only.
Private Key: Known only to you — used for decryption.

6 Ingredients of Public-Key Systems — Slide 6

  • Plaintext — الرسالة الأصلية المُدخلة للنظام

    Plaintext — the original message input to the system

  • Encryption Algorithm (خوارزمية التشفير) — تُجري تحويلات رياضية على الرسالة

    Encryption Algorithm — performs mathematical transformations on the plaintext

  • Public Key (PU) — يُستخدم كمدخل للتشفير أو التحقق من التوقيع

    Public Key (PU) — used as input for encryption or signature verification

  • Private Key (PR) — يُستخدم لفك التشفير أو إنشاء التوقيع

    Private Key (PR) — used for decryption or creating signatures

  • Ciphertext — ناتج التشفير — يبدو عشوائياً لمن لا يملك المفتاح

    Ciphertext — encrypted output — appears random to anyone without the key

  • Decryption Algorithm (خوارزمية فك التشفير) — تُعيد الرسالة الأصلية باستخدام المفتاح الخاص

    Decryption Algorithm — recovers the original plaintext using the private key

The RSA Algorithm — Core — Slides 20–22

طوّرها Rivest, Shamir, Adleman في MIT عام ١٩٧٧. المعادلتان اللتان لازم تحفظهم:

Developed by Rivest, Shamir, Adleman at MIT in 1977. The two equations you must memorize:

📐 معادلة التشفير — Encryption
📐 Encryption Formula
C = Me mod n
C = Ciphertext  |  M = Message  |  e = public exponent  |  n = modulus
📐 معادلة فك التشفير — Decryption
📐 Decryption Formula
M = Cd mod n
d = private exponent  |  proof: Med mod n = M

خطوات توليد المفاتيح — Key Setup ⭐
Key Generation Steps ⭐
📐 القوانين المستخدمة في توليد المفاتيح
📐 Formulas Used in Key Generation
n = p × q
φ(n) = (p−1)(q−1)
e: gcd(e, φ(n)) = 1  &  1 < e < φ(n)
d: d × e ≡ 1 mod φ(n)

  • اختر عددين أوليين (prime numbers) كبيرين p وq — يبقيان سريين

    Choose two large prime numbers p and q — kept secret always

  • احسب n = p × q — هذا هو الـ Modulus (المعامل) — يُنشر للجميع

    Compute n = p × q — this is the Modulus — made public

  • احسب φ(n) = (p−1)(q−1) — دالة أويلر (Euler's Totient) — تُحفظ سراً ولا تُنشر أبداً

    Compute φ(n) = (p−1)(q−1) — Euler's Totient — kept completely secret

  • اختر e بحيث: gcd(e, φ(n)) = 1 — يعني e وφ(n) لا يقسمهم عدد مشترك غير ١

    Choose e such that: gcd(e, φ(n)) = 1 — meaning e and φ(n) share no common factor except 1

  • أوجد d بحيث: d × e ≡ 1 mod φ(n) — يعني d هو المعكوس الضربي لـ e

    Find d such that: d × e ≡ 1 mod φ(n) — meaning d is the multiplicative inverse of e

  • المفتاح العام = {e, n} — انشره للجميع

    Public Key = {e, n} — share openly with everyone

  • المفتاح الخاص = {d, n} — احتفظ به سراً تاماً

    Private Key = {d, n} — keep completely secret

كل حرف له رقم من جدول: A=00, B=01, C=02, ... Z=25. الحروف تُجمع في Blocks (مجموعات) بشرط أن كل block أصغر من n.

Each letter has a code: A=00, B=01, C=02, ... Z=25. Letters are grouped into blocks, each must be smaller than n.

📐 Formula Used
Ci = Mie mod n   (for each block i)
Condition: every block Mi must be < n

✦ Example: Encrypt {ZOBGAC} — e=7, n=25121
1

Convert letters to numbers using the table
Z=25, O=14, B=01, G=06, A=00, C=02
2

Square and Multiply — Efficient Method — Slides 26–28

المشكلة: حساب 88⁷ مباشرة يُنتج رقماً ضخماً. الحل: نُقلّل بعد كل خطوة باستخدام خاصية الموديولار:

Problem: computing 88⁷ directly gives an astronomically large number. Solution: reduce after each step using the modular property:

📐 خاصية الموديولار الأساسية
📐 Core Modular Property
[ (a mod n) × (b mod n) ] mod n = (a × b) mod n
📐 قانون Square and Multiply
📐 Square and Multiply Rule
Bit = 0 → Square only: c = c² mod n
Bit = 1 → Square + Multiply: c = (c² mod n) × M mod n
✦ مثال: 5¹¹ mod 209
✦ Example: 5¹¹ mod 209
1
حوّل الأس e=11 إلى ثنائي
Convert exponent e=11 to binary
11 = 8+2+1 → binary: 1011
Bits: 1 → 0 → 1 → 1
2
بت 1 (الأول دائماً 1): ابدأ بـ c = M
Bit 1 (first bit always 1): start with c = M
c = 5
c = 5
3
بت 0: رَبِّع فقط (Square only)
Bit 0: Square only
c = c² mod n = 5² mod 209 = 25 mod 209
c = 25
4
بت 1: رَبِّع ثم اضرب في M (Square then Multiply)
Bit 1: Square then Multiply by M
c² = 25² = 625 → 625 mod 209 = 207
c = 207 × 5 = 1035 → 1035 mod 209 = 160

📐 One-Way Function — دالة أحادية الاتجاه
📐 One-Way Function

مثال ٢ — اللوغاريتم المنفصل (Discrete Logarithm):

Example 2 — Discrete Logarithm Problem:

y = xᵏ mod n — سهل الحساب · إيجاد x من y صعب بدون k
y = xᵏ mod n — easy to compute · finding x from y is hard without k

الهجمات الرياضية الثلاثة — Slide 34

Three Mathematical Attacks — Slide 34

  • حلّل n → احسب φ(n) → احسب d

    Factor n → compute φ(n) → compute d

  • أوجد φ(n) مباشرة بدون معرفة p وq → احسب d

    Find φ(n) directly without knowing p and q → compute d

  • أوجد d مباشرة بدون معرفة φ(n)

    Find d directly without knowing φ(n)

🔑 لماذا كلها تفشل؟
🔑 Why do all three fail?

كل الهجمات تعتمد على تحليل n — وهذا مستحيل حسابياً لـ n كبير (1024 بت فأكثر). كلما زاد حجم n، زادت الصعوبة أضعافاً.

All attacks ultimately rely on factoring n — computationally infeasible for large n (1024+ bits). Every extra bit roughly doubles the difficulty.

الجذر الأولي Primitive Root — Slide 4
Primitive Root Background — Slide 4

Primitive Root (الجذر الأولي): العدد α يكون جذراً أولياً للعدد الأولي q إذا أنتجت Y = αˣ mod q قيماً مختلفة لكل X من 1 إلى q−1.

Primitive Root: α is a primitive root of prime q if Y = αˣ mod q produces a different value for every X from 1 to q−1.

✦ مثال: هل α=3 جذر أولي لـ q=5؟
✦ Example: Is α=3 a primitive root of q=5?
1
القانون: احسب Y = 3ˣ mod 5 لكل X من 1 إلى 4
Formula: compute Y = 3ˣ mod 5 for each X from 1 to 4
X=1 → 3¹ mod 5 = 3
X=2 → 3² mod 5 = 9 mod 5 = 4
X=3 → 3³ mod 5 = 27 mod 5 = 2
X=4 → 3⁴ mod 5 = 81 mod 5 = 1
النواتج {1,2,3,4} — كلها مختلفة ✓
إذن α=3 هو جذر أولي لـ q=5
Therefore α=3 is a primitive root of q=5

ما هي دالة الـ Hash؟ — Slides 3–6
What is a Hash Function? — Slides 3–6

دالة تأخذ رسالة بأي حجم وتُنتج قيمة ثابتة الحجم تُسمى Hash Value أو Message Digest. الهدف الأساسي: سلامة البيانات (Data Integrity).

A function that takes any-size message and produces a fixed-size output called a Hash Value or Message Digest. Primary purpose: Data Integrity.

📐 المعادلة الأساسية
📐 Core Formula
h = H(M)
H = hash function · M = message (any size) · h = hash value (fixed size)
💡 خصائص Hash الأساسية
💡 Core Hash Properties

١- Variable input: تقبل أي حجم رسالة
٢- Fixed output: ناتجها دائماً نفس الحجم (مثلاً 256 بت في SHA-256)
٣- One-way: لا يمكن استرداد M من h
٤- Deterministic: نفس M تُعطي دائماً نفس h

1- Variable input: accepts any message size
2- Fixed output: always produces same size output (e.g. 256 bits in SHA-256)
3- One-way: cannot recover M from h
4- Deterministic: same M always gives same h

🔑 الفرق بين Second Preimage وCollision — يجي في الاختبار
🔑 Second Preimage vs Collision — appears in exams

Second Preimage: عندك M محدد وتحاول تجد M' يعطي نفس الـ hash
Collision: تختار M وM' بحرية وتحاول تجد أي زوج يعطي نفس الـ hash (أسهل)

Second Preimage: You have a specific M and try to find M' with the same hash
Collision: You freely choose both M and M' and try to find any pair with same hash (easier)
→ Collision Resistant is stronger than Second Preimage Resistant

١. هجوم القوة الغاشمة Brute-Force Attack

1. Brute-Force Attack

يجرّب قيماً عشوائية حتى يجد تصادماً. القوة تعتمد فقط على طول الـ hash بالبت.

Tries random values until a collision is found. Strength depends only on the hash length in bits.

📐 جهد البحث عن Collision — Birthday Paradox
📐 Collision Search Effort — Birthday Paradox
Collision needs ≈ 2n/2 operations (n = hash bits)
Preimage needs ≈ 2n operations
Example: n=64 → collision ≈ 232 ≈ 4 billion tries
Example: n=256 → collision ≈ 2128 → practically impossible

٢. Birthday Attack — Slides 18

2. Birthday Attack — Slide 18

هجوم ذكي يستغل Birthday Paradox (مفارقة عيد الميلاد): في غرفة فيها 23 شخص، احتمال أن اثنين لهم نفس يوم الميلاد هو 50٪. الفكرة: تجمع تنويعات من رسالتين وتبحث عن تطابق.

A smart attack exploiting the Birthday Paradox: in a room of 23 people, there's a 50% chance two share a birthday. The idea: collect variations of two messages and search for a match.

✦ خطوات Birthday Attack
✦ Birthday Attack Steps
1
المهاجم يُعدّ 2^(n/2) تنويعاً للرسالة الأصلية x → يُخزّن hashها
Attacker prepares 2^(n/2) variations of legitimate message x → stores their hashes
x, x', x'', ... → H(x), H(x'), H(x''), ...
2
المهاجم يُعدّ 2^(n/2) تنويعاً للرسالة المزوّرة y
Attacker prepares 2^(n/2) variations of fraudulent message y
y, y', y'', ... → H(y), H(y'), H(y''), ...
3
يبحث عن x' وy' بحيث H(x') = H(y')
Finds x' and y' such that H(x') = H(y')
H(x') = H(y') → collision found!
4
يُقدّم x' للضحية لتوقيعه (رسالة شرعية) ثم يستبدلها بـ y' (رسالة مزوّرة)
Presents x' to victim for signing (legitimate message) then replaces with y' (fraudulent)
sign(x') = sign(y') → signature is valid on fraudulent message!
⚠️ لماذا هذا خطير؟
⚠️ Why is this dangerous?

لأن H(x') = H(y') فإن نفس التوقيع يعمل على الرسالتين — المهاجم يحصل على توقيع رسالة مزوّرة دون الحاجة لمعرفة المفتاح الخاص.

Since H(x') = H(y'), the same signature works on both messages — attacker gets a valid signature on a fraudulent message without knowing the private key.
